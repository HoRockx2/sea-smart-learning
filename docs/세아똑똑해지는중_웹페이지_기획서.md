## 1. 프로젝트 개요

### 1.1 배경 및 목적

초등학교 저학년 아이가 학습 과정에서 자주 틀리는 문제 유형들을 체계적으로 반복 학습할 수 있는 웹 기반 학습 도구를 제작합니다. 수학, 음악, 국어 등 교과 전반에서 발생하는 취약 영역을 개념 설명과 연습 문제로 보강하는 것이 핵심 목표입니다.

### 1.2 핵심 가치

- **반복 학습**: 틀리기 쉬운 문제를 여러 번 연습하여 완전 학습 달성
- **즐거운 학습**: 이야기와 캐릭터를 활용한 스토리텔링 방식으로 개념을 설명하여, 아이가 자연스럽게 몰입하고 흥미를 느끼는 학습 경험 제공
- **자기 주도**: 아이가 스스로 선택하고 풀어볼 수 있는 직관적 구조
- **확장 가능**: 새로운 주제를 언제든 추가할 수 있는 유연한 설계
- **일관된 경험**: 모든 상세 페이지가 동일한 디자인 포맷을 유지

### 1.3 대상 사용자

| 구분 | 설명 |
|------|------|
| 주 사용자 | 초등학교 1~2학년 학생 |
| 보조 사용자 | 학부모 (콘텐츠 관리, 페이지 생성 요청) |
| 사용 환경 | 태블릿/패드/모바일 (터치 기반 인터랙션 최적화) |
| 사용 시나리오 | 가정 학습, 자투리 시간 활용, 시험 전 복습 |

## 2. 사이트 구조 (Information Architecture)

### 2.1 전체 구조도

사이트는 메인 페이지(허브)와 개별 상세 페이지(스포크)로 구성된 허브-스포크 구조를 따릅니다. 메인 페이지에서 모든 학습 주제에 접근할 수 있으며, 새로운 주제가 추가되면 메인 페이지의 카드 목록에 자동으로 반영됩니다.

```
                    ┌─────────────────────┐
                    │   메인 페이지        │
                    │   (index.html)       │
                    └──────┬──────┬───────┘
                           │      │
              ┌────────────┼──────┼────────────┐
              ▼            ▼      ▼            ▼
         ┌────────┐  ┌────────┐  ┌────────┐
         │  수학   │  │  음악   │  │  국어   │
         └───┬────┘  └───┬────┘  └───┬────┘
             │            │            │
        ┌────┴────┐  ┌───┴────┐  ┌───┴─────┐
        │받아올림  │  │음표 길이│  │받아쓰기  │
        │받아내림  │  │박자 세기│  │띄어쓰기  │
        │구구단    │  │음이름   │  │맞춤법    │
        │도형 이름 │  │...     │  │...      │
        │...      │  │        │  │         │
        └─────────┘  └────────┘  └─────────┘
```

### 2.2 URL 규칙

파일 기반의 단순한 구조를 사용합니다. 별도의 서버 없이 정적 파일로 호스팅 가능합니다.

| 페이지 | 경로 예시 | 설명 |
|--------|-----------|------|
| 메인 | index.html | 학습 주제 카드 목록 (허브) |
| 수학 상세 | pages/math-carrying.html | 덧셈 받아올림 학습 |
| 음악 상세 | pages/music-note-length.html | 음표 길이 학습 |
| 국어 상세 | pages/korean-spacing.html | 띄어쓰기 학습 |

### 2.3 파일 구조

```
project/
├── index.html              ← 메인 페이지
├── css/
│   └── common.css          ← 공통 스타일시트
├── js/
│   └── common.js           ← 공통 스크립트
├── assets/
│   └── icons/              ← 주제별 아이콘
├── pages/
│   ├── math-carrying.html  ← 수학: 받아올림
│   ├── music-notes.html    ← 음악: 음표
│   └── ...                 ← 새 주제 추가 시 여기에 생성
└── data/
    └── topics.json         ← 주제 목록 데이터
```

## 3. 메인 페이지 기획

### 3.1 페이지 목적

메인 페이지는 학습 허브 역할을 하며, 아이가 원하는 학습 주제를 직관적으로 선택할 수 있도록 합니다. 카드형 UI로 각 주제를 시각적으로 표현하고, 카테고리별 필터링을 지원합니다.

### 3.2 레이아웃 구성

| 영역 | 위치 | 내용 |
|------|------|------|
| 헤더 | 상단 고정 | 사이트 로고/이름, 간단한 인사말 |
| 카테고리 필터 | 헤더 아래 | 전체 / 수학 / 음악 / 국어 탭 버튼 |
| 학습 카드 그리드 | 본문 중앙 | 주제별 카드 (아이콘 + 제목 + 간단 설명) |
| 푸터 | 하단 | 저작권, 부모님 안내 링크 |

### 3.3 학습 카드 구성 요소

각 카드는 다음 요소를 포함합니다:

- **주제 아이콘**: 직관적인 이모지 또는 일러스트 (예: 수학 = 숫자 블록)
- **주제 제목**: 큰 글씨로 명확하게 (예: "받아올림 덧셈")
- **핵심 개념 미리보기**: 이 주제에서 배울 핵심 내용을 한눈에 보여주는 문구 (예: "10이 넘으면 윗자리로 올려줘요!") — 아이가 카드만 봐도 무엇을 배우는지 감을 잡을 수 있도록 함
- **스토리 한 줄**: 이야기 도입부 맛보기 (예: "토끼가 당근을 세다가 고민에 빠졌어요…")
- **카테고리 뱃지**: 수학 / 음악 / 국어 색상 구분
- **난이도 표시**: 별 아이콘으로 1~3단계 표시 (선택 사항)

### 3.4 카드 데이터 구조 (topics.json)

메인 페이지는 topics.json 파일에서 주제 목록을 읽어 카드를 동적으로 생성합니다. 각 항목에는 핵심 개념(concept)과 이야기 도입(story)이 포함되어, 카드에서 바로 학습 내용과 흥미를 전달합니다. 새 주제를 추가할 때는 이 JSON에 항목을 추가하고 해당 HTML 파일을 pages/ 폴더에 생성하면 됩니다.

```json
{
  "topics": [
    {
      "id": "math-carrying",
      "title": "받아올림 덧셈",
      "concept": "10이 넘으면 윗자리로 올려줘요!",
      "story": "토끼가 당근을 세다가 고민에 빠졌어요…",
      "category": "math",
      "icon": "🔢",
      "difficulty": 2,
      "page": "pages/math-carrying.html"
    },
    {
      "id": "music-note-length",
      "title": "음표 길이 알기",
      "concept": "음표 모양마다 소리 길이가 달라요!",
      "story": "음표 친구들이 달리기 시합을 시작했어요…",
      "category": "music",
      "icon": "🎵",
      "difficulty": 1,
      "page": "pages/music-note-length.html"
    }
  ]
}
```

### 3.5 인터랙션

- 카드 탭/클릭 시 해당 상세 페이지로 이동
- 카테고리 탭 버튼으로 필터링 (전체 / 수학 / 음악 / 국어)
- 카드에 호버(터치) 시 살짝 확대되는 애니메이션
- 태블릿/모바일 환경: 카드 2열(태블릿) / 1열(모바일) 그리드, 큰 터치 영역 확보

## 4. 상세 페이지 템플릿 기획

모든 상세 페이지는 동일한 템플릿 구조를 따릅니다. AI로 새 페이지를 생성할 때 이 템플릿을 기준으로 작성하면 일관된 디자인이 유지됩니다.

### 4.1 페이지 구조 (상단 → 하단)

| 순서 | 섹션 | 설명 | 필수 여부 |
|------|------|------|-----------|
| 1 | 상단 내비게이션 | ← 뒤로가기 버튼 + 주제 제목 | 필수 |
| 2 | 개념 설명 영역 | 핵심 개념을 그림/도표와 함께 설명 | 필수 |
| 3 | 예시 문제 | 풀이 과정을 단계별로 보여주는 예시 | 필수 |
| 4 | 연습 문제 영역 | 직접 풀어보는 인터랙티브 문제 (3~5문제) | 필수 |
| 5 | 결과 / 피드백 | 정답 확인, 점수, 캐릭터의 격려 메시지 | 필수 |
| 6 | 다시 풀기 버튼 | 새로운 문제 세트로 재도전 | 필수 |
| 7 | 관련 주제 추천 | 비슷한 주제 카드 2~3개 표시 | 선택 |

### 4.2 각 섹션 상세 정의

#### 4.2.1 상단 내비게이션

- 왼쪽: ← 아이콘 + "돌아가기" 텍스트 (메인 페이지로 이동)
- 가운데: 주제 제목 (예: "받아올림 덧셈")
- 카테고리 색상 배경 바를 상단에 표시

#### 4.2.2 개념 설명 영역 (스토리텔링 방식)

- 카드형 박스 안에 개념 설명
- **이야기 형식으로 개념 도입**: 친근한 캐릭터나 일상 속 상황을 활용하여 "왜 이걸 배우는지"부터 자연스럽게 시작 (예: "토끼 두 마리가 당근을 나눠 먹으려고 해요. 어떻게 나눌 수 있을까?")
- **캐릭터 활용**: 주제마다 등장하는 친근한 캐릭터가 아이에게 말을 걸듯 설명 (예: 수학요정, 음표친구, 글자탐험가 등)
- **생활 속 예시 연결**: 추상적 개념을 아이의 일상과 연결 (예: 받아올림 → "사탕 10개가 모이면 한 봉지!")
- 핵심 내용은 강조 색상으로 하이라이트
- 가능한 경우 시각 자료 (그림, 다이어그램) 포함
- 글씨 크기: 본문 18px 이상 (저학년 가독성)
- 문장은 짧고 쉬운 단어 사용, 말하듯 부드러운 어투

#### 4.2.3 예시 문제

- "같이 풀어볼까?" 같은 친근한 헤딩과 함께 시작
- 이야기 흐름을 이어가며 풀이 과정을 1단계, 2단계, 3단계로 분리 (예: "토끼가 먼저 일의 자리를 더해봤어요 → 어? 10이 넘었네! → 그래서 십의 자리로 1을 올려줬어요")
- 각 단계에 시각적 번호 태그 표시
- 정답 부분은 색상으로 강조

#### 4.2.4 연습 문제 영역

- 한 번에 3~5문제 제시
- 각 문제는 카드 형태로 독립적으로 표시
- 입력 방식: 주제에 따라 객관식 / 직접 입력 / 드래그앤드롭
- 큰 입력 필드와 버튼 (터치에 최적화)
- "정답 확인" 버튼은 각 문제마다 또는 전체 제출 중 선택

#### 4.2.5 결과 / 피드백

- 맞은 개수 / 전체 개수 표시 (예: 4/5)
- 정답: 초록색 체크 + 캐릭터의 칭찬 한마디 (예: "와! 토끼도 깜짝 놀랐어!")
- 오답: 올바른 답 표시 + 캐릭터의 다정한 설명 (예: "아깝다! 여기를 다시 보면…") — 좌절감 최소화
- 전체 결과 후 이야기 마무리 격려 메시지 (예: "토끼가 당근을 잘 나눠줬어요! 네 덕분이야!")

#### 4.2.6 다시 풀기

- 큰 버튼으로 "다시 풀어볼래요!" 제공
- 가능하면 문제를 랜덤으로 섞어서 제공

### 4.3 상세 페이지 HTML 템플릿 가이드

AI에게 새 페이지 생성을 요청할 때 아래 사항을 포함하여 요청하면 일관된 결과를 얻을 수 있습니다.

```
[AI 페이지 생성 요청 시 포함할 정보]

1. 주제: (예: 뺄셈 받아내림)
2. 카테고리: math / music / korean
3. 스토리 설정: (어떤 상황/캐릭터로 개념을 풀어낼지)
4. 개념 설명 내용: (아이에게 설명할 핵심 개념)
5. 예시 문제 및 풀이: (단계별 풀이 1개)
6. 연습 문제 유형: 객관식 / 직접입력 / 드래그앤드롭
7. 문제 개수: 3~5개
8. 난이도: 1~3

[필수 참조 파일]
- css/common.css (공통 스타일)
- js/common.js (공통 기능)
- 템플릿 구조 (섹션 4.1 참조)
```

## 5. 주제 확장 가이드

이 사이트는 주제를 언제든지 자유롭게 추가할 수 있도록 설계됩니다. 새로운 과목이나 학습 내용이 필요할 때 아래 절차를 따르면 됩니다.

### 5.1 새 주제 추가 절차

1. topics.json에 새 주제 항목 추가
2. pages/ 폴더에 새 HTML 파일 생성 (AI를 활용하여 템플릿 기반으로 제작)
3. 필요 시 assets/icons/에 아이콘 추가
4. 메인 페이지가 자동으로 새 카드를 표시

### 5.2 새 카테고리 추가

수학, 음악, 국어 외에 새로운 카테고리(예: 영어, 사회, 과학)를 추가할 수도 있습니다.

- topics.json의 category 필드에 새 카테고리 값 사용
- css/common.css에 새 카테고리 색상 추가
- 메인 페이지 필터 탭에 자동 반영되도록 동적 탭 생성

### 5.3 확장 예시

| 카테고리 | 추가 가능 주제 예시 |
|----------|---------------------|
| 수학 | 구구단 연습, 시계 읽기, 길이 단위 변환, 도형 이름 맞추기 |
| 음악 | 박자 세기, 음이름 읽기 (도레미), 리듬 패턴 맞추기 |
| 국어 | 받아쓰기 단어, 띄어쓰기 규칙, 비슷한 말 구분 |
| 영어 (향후) | 알파벳 순서, 기초 단어, 파닉스 발음 |
| 과학 (향후) | 계절과 날씨, 동물 분류, 식물 관찰 |
| 사회 (향후) | 지도 읽기, 우리 동네 시설, 안전 수칙 |

## 6. 디자인 시스템 (스타일 가이드)

모든 페이지에서 일관된 Look & Feel을 유지하기 위한 디자인 규칙입니다. 이 내용은 css/common.css에 반영됩니다.

### 6.1 컬러 팔레트

| 용도 | 색상 | HEX 코드 | 사용처 |

| 수학 카테고리 | 노란색 계열 | #FFC107 | 수학 카드 뱃지, 상단 바 |
| 음악 카테고리 | 초록색 계열 | #4CAF50 | 음악 카드 뱃지, 상단 바 |
| 국어 카테고리 | 분홍색 계열 | #E91E63 | 국어 카드 뱃지, 상단 바 |
| 정답 | 초록 | #27AE60 | 정답 표시, 성공 메시지 |
| 오답 | 빨강 | #E74C3C | 오답 표시, 주의 메시지 |
| 배경 | 밝은 회색 | #F5F7FA | 페이지 배경 |
| 카드 배경 | 흰색 | #FFFFFF | 카드, 문제 박스 |
| 본문 텍스트 | 진한 회색 | #333333 | 일반 텍스트 |

### 6.2 타이포그래피

| 요소 | 폰트 | 크기 | 굵기 |
|------|------|------|------|
| 페이지 제목 | Pretendard / Noto Sans KR | 28px | Bold |
| 섹션 제목 | Pretendard / Noto Sans KR | 22px | Bold |
| 본문 (개념 설명) | Pretendard / Noto Sans KR | 18px | Regular |
| 문제 텍스트 | Pretendard / Noto Sans KR | 20px | Medium |
| 버튼 텍스트 | Pretendard / Noto Sans KR | 18px | Bold |

### 6.3 컴포넌트 스타일

**카드 컴포넌트**
- 모서리 둥글기: 16px
- 그림자: 0 4px 12px rgba(0,0,0,0.08)
- 패딩: 24px
- 터치 영역: 최소 48px x 48px

**버튼 컴포넌트**
- 모서리 둥글기: 12px
- 높이: 최소 52px (터치 친화적)
- 주요 버튼: 파란색 배경 + 흰색 텍스트
- 보조 버튼: 흰색 배경 + 테두리

**입력 필드**
- 높이: 52px 이상
- 폰트 크기: 20px
- 모서리 둥글기: 8px
- 포커스 시 파란색 테두리 강조

### 6.4 반응형 기준

| 기기 | 화면 너비 | 카드 열 수 | 비고 |
|------|-----------|-----------|------|
| 태블릿/모바일 (주요) | 768px ~ 1024px | 2열 | 주요 타겟 환경 |
| 모바일 | ~ 767px | 1열 | 세로 모드 대응 |
| 데스크톱 | 1025px ~ | 3열 | 부가 지원 |

### 6.5 접근성 고려사항

- 충분한 색상 대비 (WCAG AA 기준 이상)
- 큰 글씨와 터치 영역 (저학년 손가락 크기 고려)
- 명확한 피드백 (소리, 색상, 텍스트 동시 제공)
- 심플한 내비게이션 (최대 1단계 깊이)

### 6.6 스토리텔링 디자인 가이드

- **캐릭터 일러스트 영역**: 개념 설명과 피드백에 캐릭터 이미지를 배치할 고정 영역 확보 (왼쪽 또는 상단)
- **말풍선 UI**: 캐릭터가 말하는 형태의 말풍선 박스로 설명과 피드백을 감싸서 대화하는 느낌 제공
- **이야기 흐름 표시**: 개념 설명 → 예시 → 연습 순서가 하나의 이야기처럼 연결되도록 섹션 간 연결 문구 배치 (예: "이제 네가 해볼 차례야!")
- **카테고리별 캐릭터 예시**: 수학 = 숫자를 좋아하는 캐릭터, 음악 = 음표 모양 캐릭터, 국어 = 글자 탐험가 캐릭터 (이모지 또는 간단한 일러스트)

## 7. 기능 요구사항

### 7.1 메인 페이지 기능

| 기능 | 우선순위 | 설명 |
|------|----------|------|
| 카드 목록 표시 | 필수 | topics.json에서 주제를 읽어 카드 그리드로 표시 |
| 카테고리 필터 | 필수 | 탭 버튼으로 카테고리별 필터링 |
| 카드 링크 | 필수 | 카드 클릭 시 해당 상세 페이지로 이동 |
| 동적 카테고리 탭 | 권장 | topics.json의 카테고리를 기반으로 탭 자동 생성 |
| 로딩 애니메이션 | 선택 | 카드 로딩 시 부드러운 전환 효과 |

### 7.2 상세 페이지 기능

| 기능 | 우선순위 | 설명 |
|------|----------|------|
| 개념 설명 렌더링 | 필수 | 텍스트 + 이미지 기반 개념 설명 표시 |
| 연습 문제 출제 | 필수 | 인터랙티브 문제 표시 (객관식/입력) |
| 정답 확인 | 필수 | 제출 후 즉시 정답/오답 피드백 |
| 점수 표시 | 필수 | 맞은 개수 / 전체 개수 표시 |
| 캐릭터 피드백 | 필수 | 정답/오답 시 캐릭터의 격려 메시지 |
| 다시 풀기 | 필수 | 문제 초기화 및 재도전 |
| 문제 랜덤 | 권장 | 다시 풀기 시 문제 순서 섞기 |
| 관련 주제 추천 | 선택 | 페이지 하단에 비슷한 주제 카드 표시 |

### 7.3 기술 스택 (권장)

| 영역 | 기술 | 이유 |
|------|------|------|
| 마크업 | HTML5 | 별도 빌드 도구 불필요, 정적 호스팅 가능 |
| 스타일 | CSS3 (common.css) | 일관된 디자인 유지 |
| 스크립트 | Vanilla JavaScript | 프레임워크 의존성 없이 가볍게 운영 |
| 데이터 | JSON (topics.json) | 주제 목록 관리 간편 |
| 호스팅 | GitHub Pages / Netlify | 무료, 간편한 배포 |

## 8. 향후 계획

### 8.1 단기 (1~2주)

- 메인 페이지 디자인 및 개발
- 공통 CSS/JS 파일 작성
- 상세 페이지 템플릿 HTML 작성
- 첫 번째 콘텐츠 3개 제작 (수학 1, 음악 1, 국어 1)

### 8.2 중기 (1~2개월)

- 주제별 콘텐츠 확충 (각 카테고리 5개 이상)
- 사용성 테스트 (아이와 함께 테스트)
- 피드백 반영 및 UI 개선
- 스토리텔링 콘텐츠 고도화 (캐릭터 일러스트, 이야기 다양화)

### 8.3 장기 (향후)

- 새 카테고리 추가 (영어, 과학, 사회)
- 학습 진도 저장 기능 (로컬 스토리지 활용)
- 틀린 문제 모아보기 기능
- 난이도별 문제 자동 추천
- 학년이 올라감에 따른 콘텐츠 난이도 확장
